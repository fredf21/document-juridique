# Technical Specification - US04: Interaction Vocale (TTS)

## 1. Overview
The goal is to enhance user accessibility and provide a "premium" experience by allowing the system to read the AI-generated summary aloud.

## 2. Technical Stack
- **Engine**: `gTTS` (Google Text-to-Speech) - Python library wrapping the Google TTS API.
- **Integration**: Streamlit native `st.audio` component.
- **Service Layer**: `src/services/audio_service.py` [NEW].

## 3. Architecture & Service Design
### `src/services/audio_service.py`
The service must implement a robust function for text-to-audio conversion:
- **Function**: `generate_audio_bytes(text: str) -> BytesIO`
- **Logic**:
    1. Initialize `gTTS` object with `text` and `lang='fr'`.
    2. Save audio data into a `BytesIO` buffer (in-memory) to avoid disk I/O and temporary file management issues.
    3. Return the buffer's content.

## 4. UI/UX Integration (`src/app.py`)
- **Placement**: A button labelled "🔊 Écouter le résumé" placed immediately after the summary text.
- **Feedback**: While the audio is being generated, show a subtle loading indicator (spinner) within the button context or a small message.
- **Player**: Use `st.audio` with the generated bytes. The player should only appear once the audio is ready.

## 5. Performance & Reliability
- **Caching**: Wrap the service call with `@st.cache_data`. This is critical to:
    - Reduce latency for repeated playbacks.
    - Avoid unnecessary calls to Google's servers.
- **Error Handling**:
    - If Google TTS is unavailable (offline or quota), display a fallback warning: "⚠️ Le service vocal est temporairement indisponible."
    - Ensure empty or extremely short summaries don't trigger the engine.

## 6. Implementation Steps
1. Add `gTTS` to `requirements.txt`.
2. Create `src/services/audio_service.py`.
3. Update `src/app.py` to import the service and add the "Écouter" button.
4. Test with various summary lengths and ensure accents are correctly voiced.
